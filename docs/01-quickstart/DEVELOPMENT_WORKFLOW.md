# 🚀 ПРАВИЛЬНЫЙ WORKFLOW РАЗРАБОТКИ

## 📋 Краткая версия:

```
1. Делаешь изменения
2. Тестируешь локально
3. Всё работает? → Push!
```

## 📚 Подробные инструкции:

### Шаг 1: Запусти локальный сервер

```powershell
cd C:\goooog
.\start-local-server.ps1
```

### Шаг 2: Открой в браузере

```
http://localhost:8000/tamagotchi-game.html
```

### Шаг 3: Тестируй (используй checklist!)

Открой: `docs\LOCAL_TESTING_CHECKLIST.md`

### Шаг 4: Только если всё работает → Push

```bash
git add .
git commit -m "описание изменений"
git push origin main
```

---

## 🎯 Золотые правила:

1. ✅ **ВСЕГДА** тестируй локально перед push
2. ✅ **ВСЕГДА** проверяй Console (F12) на ошибки
3. ✅ **ВСЕГДА** тестируй в Incognito mode
4. ❌ **НИКОГДА** не пуши если есть ошибки в Console
5. ❌ **НИКОГДА** не пуши без тестирования

---

## 🔧 Полезные команды:

### Git:
```bash
git status              # Что изменилось?
git diff                # Посмотреть изменения
git log --oneline -5    # Последние коммиты
```

### Локальный сервер:
```powershell
.\start-local-server.ps1    # Запустить
Ctrl + C                     # Остановить
```

### DevTools:
```
F12                     # Открыть DevTools
Ctrl + Shift + N        # Incognito mode
Ctrl + F5               # Hard refresh
```
