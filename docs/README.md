## üìù Documentation Standards

### File Naming Convention:
- **UPPERCASE_WITH_UNDERSCORES.md** - Main documents
- **lowercase-with-dashes.md** - Utility files
- **Category prefix** - e.g., `CHINA_`, `ADMIN_`, `SLOTS_`

### Update Protocol:
1. Update the document
2. Update `Last Updated` date in header
3. Update this README if structure changes
4. Run commit with proper message

### Archival Policy:
- Outdated docs ‚Üí Move to `.archive/`
- Keep development history in `.docs/`
- Only current, relevant docs in `docs/`
