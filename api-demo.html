üóÑÔ∏è Database Structure

üì± Table: leaderboard

Stores Telegram user data

CREATE TABLE leaderboard ( telegram_id BIGINT PRIMARY KEY, tama NUMERIC DEFAULT 0, level INTEGER DEFAULT 1, pet_name TEXT, telegram_username TEXT, linked_wallet TEXT, -- If linked to wallet created_at TIMESTAMP );

üíª Table: wallet_users

Stores Wallet user data (non-Telegram users)

CREATE TABLE wallet_users ( id BIGSERIAL PRIMARY KEY, wallet_address TEXT UNIQUE NOT NULL, user_id TEXT UNIQUE NOT NULL, username TEXT DEFAULT 'Player', tama_balance NUMERIC DEFAULT 0, level INTEGER DEFAULT 1, clicks INTEGER DEFAULT 0, telegram_id BIGINT, -- If linked to Telegram game_state JSONB, created_at TIMESTAMP );

üîó Function: get_unified_leaderboard()

Combines both tables into unified ranking

WITH combined_users AS ( -- Telegram users (non-linked) SELECT 'tg_' || telegram_id as user_id, tama as tama_balance, ... FROM leaderboard WHERE linked_wallet IS NULL UNION ALL -- Wallet users SELECT user_id, tama_balance, ... FROM wallet_users ) SELECT ROW_NUMBER() OVER (ORDER BY tama_balance DESC) as rank, user_id, username, tama_balance, level FROM combined_users;
